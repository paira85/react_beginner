# 상태 관리

# 단일 컴포넌트 안에서 useState만 사용해도 충분하지만,
# 카운터가 늘어나거나, 입력창에 글자가 들어가는 정도라면 복잡한 구조가 필요 없다.

하지만 애플리케이션의 규모가 조금만 커져도 상태 관리하기에 무리 혹은 문제가 발생한다.
EX
1. 로그인 성공 후 서버로부터 받아온 유저의 이름이나 이메일 표시
2. 댓글 작성할 때 로그인 정보, 
3. 파이페이지 프로필

Zustand : 전역상태관리
1. 애플리케이션 전체에서 공유되는 공통 뎅티ㅓ를 하나의 저장소에 모아두고, 필요한 컴포넌트가 그 저장소에 직접 접근하도록 하는 방법이다.

npm install zustand
